Capture Automation

First workflow: browser to Rasbhari

The browser extension is the first direct capture client. It will connect to any Rasbhari instance using a base URL and API key, sync your browser actions and rules, and either ask first or trigger the mapped Rasbhari action directly depending on the configured trigger mode.

1
Connect
Set the Rasbhari URL and API key inside the extension and test the connection.
2
Sync
Fetch your browser actions and rules from Rasbhari so the extension stays generic and your instance stays the source of truth.
3
Trigger
Use browser context to trigger Activities, project updates, or quick-log flows while keeping the event bus central.
Start by defining your extension-visible actions

Use Browser Actions to create the concrete Rasbhari actions the extension can later sync and expose. This is where generic browser verbs like save_current_page or capture_selection become your own Activity, event, or project-update flow.

Install Flow

How to try it right now

  1. Download rasbhari-chrome-extension.zip from this page.
  2. Open chrome://extensions and enable Developer mode.
  3. Unzip the bundle and use Load unpacked to add the extension.
  4. Open the extension’s setup page and save your Rasbhari base URL and API key.
  5. Use the extension against /automation/api/extension/status, /sync, and /execute while the richer client behaviors land next.
